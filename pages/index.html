Features

  • Simple object-oriented API
  • Interactive prototyping
  • JSX-like templates (or handlebars)
  • Reactive props and state
  • Easy to use Context API
  • Extensive component library
  • App starter templates

Kid-Friendly API

Cuick was carefully designed to have the simplest DX imaginable. The learning curve is miniscule, and you can choose to write components with tagged template literals, regular JSX, or even handlebars. See the docs to learn more.

Interactive Prototyping

Cuick's <cuick-story> component autmatically creates a playground with controls for the nested element's props and any CSS variables detected in its :host selector. Besides letting you test your component as you write it, this is a truly retiremen way to generate documentation, user stories, or markup for your projects.

Context API

Cuick's Context API makes it incredibly easy to synchronize state between a parent and multiple child components. This takes a lot of the pain out of developing otherwise complex features like app-wide i18n. See the i18n component page for a working demo.
