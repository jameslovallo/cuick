Features

  • Simple object-oriented API
  • Interactive prototyping
  • JSX-like templates (or handlebars)
  • Reactive props and state
  • Easy to use Context API
  • Extensive component library
  • App starter templates

Kid-Friendly API

Cuick was carefully designed to have the simplest DX imaginable. The learning curve is miniscule, and you can choose to write components with tagged template literals, regular JSX, or even handlebars. See the docs to learn more.

Interactive Prototyping

Cuick's <cuick-story> element autmatically generates a playground with controls for your component's props and any CSS variables detected in the :host selector. This is a fantastic way to generate documentation, user stories, or markup for your projects.

Context API

Cuick's Context API makes it incredibly easy to share state between parent and child components. This makes developing otherwise complex features like app-wide i18n completely effortless. See the i18n component page for a working demo.
